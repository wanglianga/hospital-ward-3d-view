# 医院病区三维视图系统

基于 Three.js 的医院病区 3D 可视化管理系统，直观展示病区空间布局、床位实时状态和走廊转运压力。

## 原始需求

> 用 Three.js 做一个医院病区三维视图，展示病房、护士站、走廊、电梯、隔离区、抢救室和清洁间。床位用不同颜色表达空床、占用、待清洁、隔离、转科待转和抢救备用，点击床位显示患者照护强度、隔离标记、预计转运时间、陪护情况和清洁状态。护士站能从空间视角看到哪些床位即将释放，转运员查看从当前病房到检查室或目标病区的路线，保洁人员看到待清洁床位和优先级。视图核心是把床位状态和走廊转运压力讲清楚。
> 增加隔离路线限制。隔离患者转运时，视图标出专用电梯、避开的公共走廊和需要消杀的路径，普通患者路线不能重叠。
> 加入床位清洁倒计时。患者出院后，床位进入待清洁，三维床位上显示剩余清洁时间，护士能判断何时可接收新患者。

## 功能特性

- **3D 病区场景**：完整展示病房、护士站、走廊、普通电梯、隔离专用电梯、隔离区、抢救室、清洁间的空间布局
- **6 种床位状态**：空床(绿)、占用(红)、待清洁(橙)、隔离(紫)、转科待转(蓝)、抢救备用(黄)
- **床位详情**：点击床位查看患者照护强度、隔离标记、预计转运时间、陪护情况、清洁状态
- **床位清洁倒计时**：待清洁床位在三维视图上实时显示剩余清洁时间和进度条，颜色根据紧急程度变化，护士可直观判断何时可接收新患者
- **隔离路线限制**：
  - 隔离患者自动走隔离专用路线（紫色），普通患者走普通路线（蓝色），两条路线不重叠
  - 视图标出隔离专用电梯，带紫色发光圆环和标识
  - 标出隔离患者需要避开的公共走廊区域（红色警示）
  - 标出隔离患者转运后需要消杀的路径区域（橙色动态粒子）
- **多角色视角切换**：
  - 全局视角：总览病区布局和所有床位状态
  - 护士站视角：高亮显示 2 小时内即将释放的床位
  - 转运员视角：走廊转运压力热力图 + 床位到检查室的转运路线规划（支持隔离路线）
  - 保洁视角：待清洁床位按优先级高亮排序显示，含清洁倒计时
- **走廊转运压力可视化**：热力图颜色 + 动态粒子展示走廊不同区域转运压力等级
- **转运路线规划**：可视化显示从床位到检查室的最优路径和预计时间，隔离路线额外增加消杀时间预估

## 技术栈

- **前端框架**：React 18 + TypeScript
- **构建工具**：Vite 6
- **3D 渲染**：Three.js + @react-three/fiber + @react-three/drei
- **样式方案**：Tailwind CSS 3
- **状态管理**：Zustand
- **图标库**：Lucide React
- **部署**：Docker + Nginx

## 启动方式

### 前置要求

- Node.js >= 18
- npm 或 pnpm
- Docker & Docker Compose（可选，用于容器化部署）

### 方式一：本地开发启动（推荐用于开发调试）

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发服务

```bash
npm run dev
```

访问地址：http://localhost:5173

### 方式二：本地生产构建预览

```bash
npm run build
npm run preview
```

访问地址：http://localhost:4173

### 方式三：Docker 一键启动（推荐用于部署）

#### 1. 构建并启动服务

```bash
docker compose up --build
```

如需后台运行：

```bash
docker compose up --build -d
```

访问地址：http://localhost:8080

#### 2. 停止并清理服务

```bash
docker compose down
```

│       ├── BedDetailPanel.tsx       # 床位详情面板
│       ├── StatsCards.tsx           # 床位统计卡片

├── data/

## 操作说明

- **场景操作**：鼠标左键拖拽旋转视角，滚轮缩放，右键平移
- **床位交互**：鼠标悬停床位查看基础信息，点击床位查看详细信息
- **视角切换**：点击顶部视角按钮切换不同角色视角
- **路线规划**：在全局或转运员视角下，悬停床位点击「规划转运路线」或在详情面板中点击对应按钮
